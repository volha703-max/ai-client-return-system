# brief.md — Техническое задание на лендинг-портфолио

## Проект
Лендинг-портфолио специалиста по созданию системы возврата клиентов для косметологов под ключ.

**Что я делаю (для понимания клиента):**
Создаю сайты, квизы, чат-боты и автоматические напоминания — всё под ключ, без участия клиента.

---

## 1. Цель сайта

**Главная цель:** Получить заявку — чтобы потенциальный клиент написал в Telegram.

**Вторичная цель:** Убедить косметолога, что система возврата клиентов — это реально, понятно и не страшно.

---

## 2. Целевая аудитория

**Кто это:**
- Частные косметологи, принимают клиентов сами
- Владельцы небольших кабинетов (1–3 мастера)
- Владельцы бьюти-студий

**Портрет:**
- Женщина 25–45 лет
- Смотрит с телефона
- Не разбирается в технологиях
- Боится потратить деньги впустую
- Устала от Instagram и хаоса в мессенджерах
- Не считает себя "бизнесом" — она мастер

**Главная боль:** Клиенты приходят один раз и не возвращаются.

**Запрещённые слова:** AI, CRM, воронка, лид, автоматизация (без расшифровки), конверсия, трафик.

### Секция 2 — ПРОБЛЕМА («Узнаёшь себя?»)

**Задача:** Клиент чувствует — её понимают. Остаётся читать дальше.

**Заголовок:** «Узнаёшь себя?»

**5 болей (карточки или чеклист):**
1. Клиенты пишут в WhatsApp в 11 вечера — и пропадают
2. Сделала процедуру — и больше не вернулась
3. Нет времени напоминать каждой клиентке вручную
4. Сайта нет, или он давно устарел и ничего не приносит
5. Каждый месяц не знаешь, будет ли запись

**Итог снизу (эмоциональный):**
«Работаешь руками, а система — нет. Давай это исправим.»

### Секция 3 — РЕШЕНИЕ («Что именно я делаю»)

**Задача:** Перевести внимание с боли на конкретное решение. Показать экспертизу без технических деталей.

**Заголовок:** «Я не делаю "просто сайт". Я строю систему, которая работает за тебя 24/7»

**4 инструмента с пользой каждого (карточки):**

| Что создаю | Что это даёт тебе |
|---|---|
| Сайт-визитка | Первое впечатление, которое записывает клиентов само |
| Квиз для клиентов | Отбирает тёплых клиентов, пока ты на процедуре |
| Чат-бот в Telegram | Отвечает и берёт запись в любое время суток |
| Автоматические напоминания | Ни одного потерянного клиента, никакой ручной переписки |

### Секция 4 — КАК ЭТО РАБОТАЕТ (процесс)

**Задача:** Снять страх неизвестности. Показать понятный процесс.

**Заголовок:** «Как это происходит»

**4 шага:**
1. **Созвон** — обсуждаем твою ситуацию и клиентскую базу (30–60 минут)
2. **Сборка** — настраиваю сайт, квиз и бот под твою нишу и стиль
3. **Запуск** — система начинает работать, клиенты получают первые касания
4. **Результат** — через 2–4 недели видим первые возвращения и новые записи

**Визуально:** Шаги с номерами, разделены стрелками или линией. Появляются по очереди при скролле.

### Секция 5 — КЕЙСЫ / РЕЗУЛЬТАТЫ

**Задача:** Социальное доказательство. Самый сильный элемент доверия.

**Заголовок:** «Что происходит с записью после запуска системы»

**2–3 мини-кейса (формат: имя + город + ситуация + результат цифрой):**

> «Анна, Минск, косметолог-эстетист. Настроила квиз + чат-бот. За первый месяц — 23 заявки без рекламы. Раньше приходило 4–5 с сарафана.»

> «Марина, Киев. Бот делает запись без неё, ночью. За первый месяц вернулись 11 клиентов, которые не приходили больше года.»

**Если кейсов пока нет:**
«Набираю первые кейсы по специальной цене — 3 места осталось»

- Стоимость: от 15 000 руб.
- Кнопка: «Хочу "Старт"»

**Система** (полный пакет)
- Сайт + чат-бот + автоматические напоминания клиентам
- Полная система возврата клиентов под ключ
- Стоимость: от 35 000 руб.
- Кнопка: «Хочу полную систему»

### Секция 9 — FAQ (Частые вопросы)

**Задача:** Убрать скрытые возражения тех, кто не напишет вопрос сам.

**Заголовок:** «Вопросы, которые задают чаще всего»

**7 вопросов и ответов:**

**Я не разбираюсь в технологиях — это сложно?**
Нет. Я беру на себя всё техническое. Ты только отвечаешь на вопросы о своей работе — остальное делаю я.

**Сколько времени займёт настройка?**
Обычно 5–7 рабочих дней с момента созвона до запуска системы.

**Нужно ли мне платить за какие-то сервисы отдельно?**
Да, небольшая ежемесячная оплата за Telegram-бота — обычно 300–500 руб./мес. Всё остальное входит в стоимость.

**Как быстро увижу первые результаты?**
Первые возвращения клиентов обычно появляются через 2–4 недели после запуска.

**Что если клиенты не будут пользоваться ботом?**
Мы настраиваем систему так, чтобы она была простой и удобной. Если что-то не работает — дорабатываю бесплатно в течение месяца.

**Мне нужна реклама, чтобы это работало?**
Нет. Система работает с твоей существующей базой клиентов — возвращает тех, кто уже был у тебя.

**Можно сначала просто поговорить?**
Да! Именно для этого есть бесплатный разбор — созвон 30 минут, без обязательств.

## 4. Дизайн

### Стиль и настроение
Спокойный, чистый, профессиональный. Ориентир: Aesop, Rhode, Medik8, SkinCeuticals.
Не бьюти-блог, не агентство — специалист, которому доверяют.

### Цветовая палитра
```css
:root {
  --bg:     #F7F4EF;   /* светлый кремовый фон */
  --text:   #3A3A3A;   /* тёмно-серый текст */
  --accent: #B88A7A;   /* пудрово-бежевый акцент */
  --soft:   #E7D6CF;   /* мягкий бежевый */
  --muted:  #9A8880;   /* приглушённый текст */
  --card:   #FFFFFF;   /* белые карточки */
  --border: #EDE6DF;   /* очень светлая рамка */
}
```

Секции с другим фоном: `#F2EDE6` (чуть темнее основного)

### Шрифт
```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', system-ui, sans-serif;
```

### Типографика
- H1 (Hero): `clamp(2rem, 5vw, 4rem)`, font-weight 700, letter-spacing -1.5px
- H2 (секции): `clamp(1.5rem, 3.5vw, 2.5rem)`, font-weight 700, letter-spacing -0.5px
- Лейблы/надписи над заголовками: 10–11px, font-weight 600, letter-spacing 3px, UPPERCASE, цвет var(--accent)
- Основной текст: 16–17px, font-weight 400, line-height 1.7
- Приглушённый текст: color var(--muted)

### Карточки
```css
background: var(--card);
border: 1px solid var(--border);
border-radius: 4px;
box-shadow: 0 2px 40px rgba(184,138,122,0.08);
```

### Кнопки
```css
background: var(--accent);   /* #B88A7A */
color: #FFFFFF;
border-radius: 2px;
padding: 16px 40px;
font-weight: 600;
font-size: 15px;
/* hover: background #a57a6b, transform: translateY(-2px) */
```

### Анимации
- Scroll-reveal для каждой секции: opacity 0→1, translateY(24px→0), transition 0.65s ease
- Кнопка при наведении: translateY(-2px), плавно 0.2s
- Карточки при наведении: лёгкое поднятие box-shadow

### Запрещено
- Яркие цвета: фиолетовый, розовый, оранжевый
- Тёмный фон
- Blob-анимации и неоновые эффекты
- border-radius больше 8px для карточек
- font-weight 900
- Всплывающие окна

- Карточки пакетов — в столбец
- Текст читается без зума

## 6. Технические требования

- Один самодостаточный HTML-файл (CSS и JS внутри)
- Никаких внешних библиотек и CDN — только чистый HTML/CSS/JS
- Scroll-reveal через IntersectionObserver (нативный JS)
- Фото Ольги — в формате base64 внутри файла
- Файл: `landing-cosmetolog.html` (уже существует, будем дорабатывать)

## 7. Что НЕ делаем в MVP

- Онлайн-запись и оплата
- Слайдер с фото в Hero
- Всплывающие окна
- Отдельные страницы для каждого пакета
- Видео (на перспективу)
- A/B тест заголовков (на перспективу)

---

## 8. Ключевые рекомендации из экспертного разбора

**UX-эксперт:**
- Один заголовок Hero — конкретное обещание результата, не описание услуги
- Кнопка → Telegram, не форма
- Секция «Как это работает» — 4 шага, визуально разделённые
- Социальное доказательство ДО блока с ценами

**Маркетолог:**
- Оффер строится так: конкретный результат + срок + снятие риска
- В каждой секции — цифра или цитата (не только в блоке отзывов)
- После блока цен показать ROI: «3 вернувшихся клиента = окупаемость»

**Клиент-косметолог:**
- Никаких слов «AI», «CRM», «воронка», «лид»
- Живое фото автора — обязательно
- Цена или хотя бы диапазон — без неё не напишут
- Кнопка «Узнать как это работает» → меньше страха, чем «Оставить заявку»

# ЧАСТЬ 2 — Telegram Mini App витрины Ольги

**Что это:** Telegram Mini App — витрина специалиста по системе возврата клиентов для косметологов. Косметолог заходит, проходит квиз, видит пакеты и записывается на разбор.

**Главная цель:** получить заявку — косметолог нажимает кнопку и попадает в чат с Ольгой.

### Экран 1 — Главный (оффер)

**Что видит пользователь:**

### Экраны 2–4 — Квиз (3 вопроса)

> Все три вопроса на одном DOM-элементе, переключение через JS без перезагрузки экрана

**Что видит пользователь:**
- Прогресс-бар вверху: `Вопрос 1 из 3` → `2 из 3` → `3 из 3`
- Вопрос крупным текстом
- 3–4 варианта ответа — карточки на всю ширину, крупные (min 56px высота)
- BackButton (нативная кнопка Telegram) — возврат к предыдущему вопросу
- MainButton: **«Далее»** → на последнем вопросе: **«Узнать результат»**

**Вопросы:**

1. «Как сейчас клиентки записываются к тебе снова?»
   - Пишут сами — молодец!
   - Я напоминаю вручную каждой
   - Кто вспомнит — тот и запишется
   - Новых ищу постоянно

2. «Есть ли у тебя список клиентов, которые не приходили больше 2 месяцев?»
   - Да, слежу за этим
   - Примерно помню кто есть
   - Нет, не веду такой список
   - Даже не думала об этом

3. «Сколько клиенток из 10 возвращаются сами, без твоего участия?»
   - 8–10 из 10
   - 5–7 из 10
   - 2–4 из 10
   - 1 или никто

**Действия:**
- Выбрать вариант → вариант подсвечивается, MainButton становится активной
- HapticFeedback: `impactOccurred('light')` при выборе
- MainButton → следующий вопрос или результат
- BackButton → предыдущий вопрос (свой стек состояний в JS)

### Экран 5 — Результат квиза

**Что видит пользователь:**
- Анимация 1.5 сек «Считаем результат...» перед показом
- Персонализированный заголовок по ответам:
  - Мало возвратов: «[Имя], у тебя уходит 5–7 клиенток в месяц, которые могли бы вернуться»
  - Средне: «[Имя], есть точки роста — давай разберём вместе»
  - Хорошо: «[Имя], у тебя уже есть база — давай сделаем, чтобы она работала сама»
- Подзаголовок: что именно поможет
- Рекомендованный пакет (выделен пометкой «Рекомендую для тебя»)
- MainButton: **«Посмотреть что мне подойдёт»**

**Действия:**
- MainButton → Экран 6 (Пакеты)

### Экран 6 — Пакеты услуг

**Что видит пользователь:**
- Заголовок: «Выбери, с чего начать»
- 3 карточки вертикально:
  - **Разбор** — 3 000 руб. / бесплатно первым 3
  - **Старт** — от 15 000 руб. *(пометка: «Рекомендую»)*
  - **Система** — от 35 000 руб.
- Каждая карточка: название + 1 строка описания + цена + кнопка «Подробнее»
- MainButton: **«Записаться на разбор»** (всегда видна)

**Действия:**

## Технические требования

### Запуск
```js
WebApp.expand()                    // развернуть на весь экран
WebApp.ready()                     // вызвать как можно раньше
WebApp.disableVerticalSwipes()     // отключить жест закрытия на iOS
```

### Навигация
```js
WebApp.BackButton.show()
WebApp.BackButton.onClick(handler)
WebApp.MainButton.setText('Текст')
WebApp.MainButton.show()
WebApp.MainButton.showProgress(true/false)
```

### Персонализация
```js
const name = WebApp.initDataUnsafe.user?.first_name || ''
```

### Сохранение квиза
```js
WebApp.CloudStorage.setItem('quiz_answers', JSON.stringify(answers))
WebApp.CloudStorage.getItem('quiz_answers', callback)
```

### Тактильный отклик
```js
WebApp.HapticFeedback.impactOccurred('light')        // выбор ответа в квизе
WebApp.HapticFeedback.notificationOccurred('success') // успешная отправка
WebApp.HapticFeedback.notificationOccurred('error')   // ошибка
```

### Тема (все цвета из Telegram — не прописывать жёстко)
```css
background-color: var(--tg-theme-bg-color);
color: var(--tg-theme-text-color);
--hint: var(--tg-theme-hint-color);
--secondary-bg: var(--tg-theme-secondary-bg-color);
padding-bottom: calc(16px + var(--tg-safe-area-inset-bottom));
```

### Отправка формы
- `fetch()` на Telegram-бот (webhook)
- Проверка `initData` на сервере через HMAC
- После успеха — автосообщение от бота в чат пользователя
