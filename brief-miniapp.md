# plan-miniapp.md — План разработки Telegram Mini App витрины Ольги Жебрик

**Что это:** Telegram Mini App — витрина специалиста по системе возврата клиентов для косметологов. Косметолог заходит, проходит квиз, видит пакеты и записывается на разбор.

**Главная цель:** получить заявку — косметолог нажимает кнопку и попадает в чат с Ольгой.

### Экран 1 — Главный (оффер)

**Что видит пользователь:**
- Фото Ольги (круглое, живое, с улыбкой) — вверху по центру
- Заголовок: «Клиентка сказала "всё понравилось" — и больше не пришла. Знакомо?»
- Подзаголовок: «Помогаю косметологам выстроить систему, которая возвращает клиентов сама»
- Три цифры горизонтально: `47 косметологов` / `6 недель` / `+30% выручки`
- MainButton: **«Пройти бесплатный разбор»**

**Элементы:**
- Фото Ольги: круг 80×80px
- H1: крупный, жирный, 2–3 строки
- Подзаголовок: мельче, приглушённый цвет
- Цифры: три блока в строку, цифра + подпись
- MainButton (нативная кнопка Telegram внизу экрана)

**Действия:**
- Нажать MainButton → переход на Экран 2 (Квиз, Вопрос 1)

### Экраны 2–4 — Квиз (3 вопроса)

> Все три вопроса на одном DOM-элементе, переключение через JS без перезагрузки экрана

**Что видит пользователь:**
- Прогресс-бар вверху: `Вопрос 1 из 3` → `2 из 3` → `3 из 3`
- Вопрос крупным текстом
- 3–4 варианта ответа — карточки на всю ширину, крупные (min 56px высота)
- BackButton (нативная кнопка Telegram) — возврат к предыдущему вопросу
- MainButton: **«Далее»** → на последнем вопросе: **«Узнать результат»**

**Вопросы:**

1. «Как сейчас клиентки записываются к тебе снова?»
   - Пишут сами — молодец!
   - Я напоминаю вручную каждой
   - Кто вспомнит — тот и запишется
   - Новых ищу постоянно

2. «Есть ли у тебя список клиентов, которые не приходили больше 2 месяцев?»
   - Да, слежу за этим
   - Примерно помню кто есть
   - Нет, не веду такой список
   - Даже не думала об этом

3. «Сколько клиенток из 10 возвращаются сами, без твоего участия?»
   - 8–10 из 10
   - 5–7 из 10
   - 2–4 из 10
   - 1 или никто

**Действия:**
- Выбрать вариант → вариант подсвечивается, MainButton становится активной
- HapticFeedback: `impactOccurred('light')` при выборе
- MainButton → следующий вопрос или результат
- BackButton → предыдущий вопрос (свой стек состояний в JS)

### Экран 5 — Результат квиза

**Что видит пользователь:**
- Анимация 1.5 сек «Считаем результат...» перед показом
- Персонализированный заголовок по ответам:
  - Мало возвратов: «[Имя], у тебя уходит 5–7 клиенток в месяц, которые могли бы вернуться»
  - Средне: «[Имя], есть точки роста — давай разберём вместе»
  - Хорошо: «[Имя], у тебя уже есть база — давай сделаем, чтобы она работала сама»
- Подзаголовок: что именно поможет
- Рекомендованный пакет (выделен пометкой «Рекомендую для тебя»)
- MainButton: **«Посмотреть что мне подойдёт»**

**Действия:**
- MainButton → Экран 6 (Пакеты)

### Экран 6 — Пакеты услуг

**Что видит пользователь:**
- Заголовок: «Выбери, с чего начать»

  - **Старт** — от 15 000 руб. *(пометка: «Рекомендую»)*

**Действия:**

### Экран 10 — Форма записи

**Что видит пользователь:**
- Заголовок: «Записаться на бесплатный разбор»
- Подзаголовок: «60 минут. Конкретный план. Без давления.»
- Поле: Имя (автоматически из Telegram)
- Поле: Telegram или телефон
- Если пришли с Экрана 7 — выбранный пакет подставлен
- Под кнопкой: «Отвечу в течение 2 часов»
- MainButton: **«Отправить заявку»**

**Действия:**
- MainButton (только если поля заполнены) → отправка → Экран 11

- BackButton → предыдущий экран

**Что видит пользователь:**

## Технические требования

### Запуск
```js
WebApp.expand()                    // развернуть на весь экран
WebApp.ready()                     // вызвать как можно раньше
WebApp.disableVerticalSwipes()     // отключить жест закрытия на iOS
```

### Навигация
```js
WebApp.BackButton.show()
WebApp.BackButton.onClick(handler)
WebApp.MainButton.setText('Текст')
WebApp.MainButton.show()
WebApp.MainButton.showProgress(true/false)
```

### Персонализация
```js
const name = WebApp.initDataUnsafe.user?.first_name || ''
```

### Сохранение квиза
```js
WebApp.CloudStorage.setItem('quiz_answers', JSON.stringify(answers))
WebApp.CloudStorage.getItem('quiz_answers', callback)
```

### Тактильный отклик
```js
WebApp.HapticFeedback.impactOccurred('light')        // выбор ответа в квизе
WebApp.HapticFeedback.notificationOccurred('success') // успешная отправка
WebApp.HapticFeedback.notificationOccurred('error')   // ошибка
```

### Тема (все цвета из Telegram — не прописывать жёстко)
```css
background-color: var(--tg-theme-bg-color);
color: var(--tg-theme-text-color);
--hint: var(--tg-theme-hint-color);
--secondary-bg: var(--tg-theme-secondary-bg-color);
padding-bottom: calc(16px + var(--tg-safe-area-inset-bottom));
```

### Отправка формы
- `fetch()` на Telegram-бот (webhook)
- Проверка `initData` на сервере через HMAC
- После успеха — автосообщение от бота в чат пользователя

## Чего НЕ будет в первой версии

- Онлайн-оплата — только заявка на разбор
- Фотогалерея работ — только текстовые кейсы
- Личный кабинет — без истории заявок
- Программа лояльности и бонусы
- Видео-отзывы
- Горизонтальные карусели и слайдеры
- Сложные анимации переходов — только fade
- Отдельная страница FAQ
- A/B тест формулировок

## Что тестировать перед запуском

1. iOS — жесты, клавиатура, safe area
2. Android бюджетный — производительность
3. Telegram Desktop — layout без touch
4. Переключился в чат → вернулся: ответы квиза сохранены?
5. Клавиатура в форме → MainButton не перекрывает поля?
6. Нет интернета при отправке → данные не потеряны?
7. Тёмная тема → все тексты читаемы?
8. Время до первого контента → не более 2 секунд
